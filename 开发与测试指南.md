# vLLM v0.15.1 开发与测试指南

本文面向"以学习/二次开发为目的"使用本分支（`v0.15.1`）的同学，给出从零搭建可开发、
可调试、可测试环境的完整步骤。配合 `example.py`（最小推理示例）、`architecture.md`、
`core_concepts.md` 一起看效果更好。

> 选这个版本的理由见 `README.md`：它是"纯 V1 架构、MRV2 尚未对常规 LLM 全面生效"
> 的甜蜜区间，worker 层仍是单文件 `gpu_model_runner.py`，调度器默认同步，
> 心智负担小，最接近从 nano-vllm 过渡的需求。

---

## 0. 前置要求

- **Python**：3.10 ~ 3.13（`pyproject.toml` 里 `requires-python = ">=3.10,<3.14"`，推荐 3.11/3.12）
- **操作系统**：Linux（GPU 路径）；CPU/macOS 也能跑用于读代码与调试
- **Git**：用于源码安装
- **[uv](https://docs.astral.sh/uv/)**：用它管理虚拟环境与依赖（比 pip/conda 快很多）

安装 uv（已装可跳过）：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
# 或：pip install uv
```

用 uv 创建并激活独立虚拟环境（uv 会自动下载对应的 Python 解释器）：

```bash
uv venv ~/.venv/vllm-dev --python 3.12
source ~/.venv/vllm-dev/bin/activate
```

> 下文所有安装命令都用 `uv pip ...`，效果与 `pip` 一致但更快；只要虚拟环境已激活，
> `uv pip` 会自动装到该环境里。

## 2. 安装方式（按硬件三选一）

开发阶段一律用**可编辑安装**（`-e`），这样改 Python 代码无需重装即可生效。

### 2A. 有 NVIDIA GPU（推荐，能真正跑推理）

最省事的方式是先装好与 vLLM 匹配的 PyTorch（CUDA 版），再可编辑安装 vLLM：

```bash
# 1) 先装 CUDA 版 PyTorch（按你的 CUDA 选 cu121/cu124 等，参考 pytorch.org）
uv pip install torch --torch-backend=cu124

# 2) 让 vLLM 复用已装好的 torch，避免被重新拉一个版本
python use_existing_torch.py

# 3) 可编辑安装（只编译 Python 侧，复用预编译 kernel）
VLLM_USE_PRECOMPILED=1 uv pip install -e . -v
```

- `VLLM_USE_PRECOMPILED=1`：下载官方预编译的 C++/CUDA 扩展，**跳过本地编译**，
  几分钟即可装好——适合"只改 Python 层"的学习/调试场景。
- 若你要改 CUDA/C++ kernel，则去掉该变量做完整源码编译（需要 CUDA toolkit、`ninja`，
  编译耗时较长）。详见 `docs/getting_started/installation/gpu.cuda.inc.md`。

### 2B. 无 GPU，仅 CPU（适合纯读代码 / 跑单测 / 调试控制流）

CPU 后端需要源码编译，参考 `docs/getting_started/installation/cpu.md`：

```bash
uv pip install -r requirements/cpu-build.txt
uv pip install -r requirements/cpu.txt
VLLM_TARGET_DEVICE=cpu uv pip install -e . -v
```

CPU 路径性能很低，但足以单步跟踪 `EngineCore → Scheduler → KVCacheManager →
GPUModelRunner`（CPU 上是对应的 CPU runner）的整条调用链。

### 2C. 只想读源码、不跑推理（最快）

如果暂时不需要真正执行模型，只想让 import 跑通、用 IDE 跳转/调试：

```bash
python use_existing_torch.py
VLLM_USE_PRECOMPILED=1 uv pip install -e . -v   # 或装 CPU torch 后同样可 import
```

## 4. 验证安装

```bash
# 4.1 import 能跑通、版本正确
python -c "import vllm; print(vllm.__version__)"

# 4.2 跑最小推理示例（需先准备本地模型，见下）
huggingface-cli download Qwen/Qwen3-0.6B --local-dir ~/model/Qwen3-0.6B
python example.py
# 或指定任意模型：
VLLM_EXAMPLE_MODEL=~/model/Qwen3-0.6B python example.py
```

`example.py` 演示了两种 chat 推理写法：手动 `apply_chat_template + llm.generate`
（与 nano-vllm 完全对应）和 `llm.chat`（vLLM 自带、更简洁）。注意真实 vLLM 的
`generate` 返回 `RequestOutput` 对象，取文本用 `output.outputs[0].text`，
而 nano-vllm 返回的是 `dict`，用 `output["text"]`。

> 显存不足时：用更小的模型（如 Qwen3-0.6B），并保持 `enforce_eager=True`；
> 还可加 `gpu_memory_utilization=0.6`、`max_model_len=2048` 等限制。

## 5. 运行测试

测试配置在 `pyproject.toml` 的 `[tool.pytest.ini_options]`，测试代码在 `tests/`。

```bash
# 跑单个文件 / 单个用例（开发时最常用）
pytest tests/test_sampling_params.py -v
pytest tests/path/to/test_file.py::test_name -v

# 按关键字筛选
pytest tests/ -k "scheduler and not gpu" -v

# 并行加速（pytest-xdist，已在 test 依赖里）
pytest tests/some_dir -n auto
```

提示：
- vLLM 的整套测试很大且大量用例需要 GPU/下载模型，**不建议全量 `pytest tests/`**。
  学习时挑与你正在读的模块相关的小目录跑即可，例如：
  - 调度器：`tests/v1/core/`
  - KV cache / block pool：`tests/v1/core/`、`tests/v1/kv_cache/`（按实际目录为准）
  - 引擎/采样：`tests/v1/engine/`、`tests/samplers/`
- 用 `--collect-only` 先看某目录能收集到哪些用例：
  ```bash
  pytest tests/v1/core --collect-only -q
  ```

## 7. 推荐的源码阅读 / 调试路线

结合 `README.md` 的结论，建议按下面顺序读，并用 `example.py` 做断点入口：

1. `vllm/entrypoints/llm.py`：`LLM.generate` / `LLM.chat`——用户入口，从这里下断点。
2. `vllm/v1/engine/`：`core.py`(EngineCore) + `llm_engine.py` / `async_llm.py`——引擎主循环。
3. `vllm/v1/core/sched/scheduler.py`：统一调度（不分 prefill/decode），对标 nano-vllm 的 `Scheduler`。
4. `vllm/v1/core/kv_cache_manager.py` + `block_pool.py`：分页 KV cache / 前缀缓存，对标 `BlockManager`。
5. `vllm/v1/worker/gpu_model_runner.py`：输入准备 → forward → 采样，对标 nano-vllm 的 `ModelRunner`。

调试技巧：
- 先用 `enforce_eager=True` 关掉 CUDA Graph，调用栈更干净、可单步。
- 用单条 prompt + `max_tokens=8` 缩短一轮 decode，方便观察一个完整 step。
- 设 `VLLM_LOGGING_LEVEL=DEBUG` 打开详细日志：
  ```bash
  VLLM_LOGGING_LEVEL=DEBUG python example.py
  ```
- 想看分布式/多进程通信可暂时跳过（`tensor_parallel_size=1` 时是单进程，最易调试）。

遇到带 `model_runner_v2` / MRV2 字样的文件或 flag，按 README 建议**先跳过**，
等主线打通后再作为增量补充阅读。

## 8. 常见问题

- **装的时候又拉了一个 torch / 版本冲突**：先 `python use_existing_torch.py` 再安装，
  让 vLLM 复用你已装好的 torch。
- **本地编译太慢/失败**：加 `VLLM_USE_PRECOMPILED=1` 用预编译扩展（仅改 Python 时适用）。
- **跑 `example.py` 报模型找不到**：先用 `huggingface-cli download` 下到本地，
  并用 `VLLM_EXAMPLE_MODEL` 指向该目录。
- **显存 OOM**：换更小模型、加 `gpu_memory_utilization`、减小 `max_model_len` / `max_num_seqs`。
- **CPU 上跑很慢**：正常现象，CPU 后端只为读代码/调试控制流，不追求性能。
